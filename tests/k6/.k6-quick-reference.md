# K6 Quick Reference

## Installation
```bash
# macOS
brew install k6

# Verify
k6 version
```

## Quick Start

**1. Start the API:**
```bash
cd backend
rails server -p 3000
```

**2. Run tests:**
```bash
# Basic load test (2.5 min)
k6 run tests/k6/api-load-test.js

# Workflow test (4 min)
k6 run tests/k6/workflow-test.js

# Stress test (9 min)
k6 run tests/k6/dashboard-stress-test.js
```

## Test Comparison

| Test | Duration | Max VUs | Focus | Best For |
|------|----------|---------|-------|----------|
| api-load-test | 2.5 min | 50 | GET endpoints | Quick validation |
| workflow-test | 4 min | 10 | Full lifecycle | State transitions |
| dashboard-stress-test | 9 min | 100 | Breaking point | Capacity planning |

## Success Criteria

### api-load-test
- ✅ p95 < 500ms
- ✅ Errors < 1%

### workflow-test
- ✅ 95% workflow completion
- ✅ p95 < 5s total

### dashboard-stress-test
- ✅ p50 < 1s
- ✅ p90 < 3s
- ✅ Errors < 10%

## Custom Runs

```bash
# Change API URL
BASE_URL=http://production.com k6 run tests/k6/api-load-test.js

# Save results
k6 run --out json=results.json tests/k6/api-load-test.js

# Quiet mode
k6 run --quiet tests/k6/api-load-test.js

# Summary only
k6 run --summary-export=summary.json tests/k6/api-load-test.js
```

## Troubleshooting

**Connection refused:**
```bash
curl http://localhost:3000/api/users
# If this fails, API isn't running
```

**High error rates:**
```bash
# Check backend logs
cd backend
tail -f log/development.log
```

**Slow responses:**
```bash
# Run isolated test with 1 VU
k6 run --vus 1 --iterations 10 tests/k6/api-load-test.js
```
